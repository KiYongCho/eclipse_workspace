# package : oop.modeling.livingthing

# OOP 모델링 실습
# 아래 요구사항에 맞도록 인터페이스, 추상클래스, 클래스들을 설계해 봅니다.

# 생물은 동물과 식물로 나뉜다.
# 모든 동물은 숨을 쉰다. 먹는다. 잔다.
# 모든 식물은 숨을 쉰다.
# 모든 동물과 식물은 이름과 나이가 있다.
# 초식 동물은 풀을 먹는다.
# 육식 동물은 고기를 먹는다.
# 육식동물에는 호랑이와 사자가 있다.
# 초식동물에는 소와 사슴이 있다.
# 사람은 육식동물이면서 초식동물이다.

# 모델링이 끝나면 동료와 모델링한 결과에 대해 토론해 봅니다.

# 생물	LivingThing
# 동물	Animal
# 식물	Plant
# 숨을 쉬다	breathe
# 먹다	eat
# 자다	sleep
# 이름	name
# 나이	age
# 초식동물	Herbivore
# 육식동물	Carnivore
# 풀	grass
# 고기	meat
# 호랑이	Tiger
# 사자	Lion
# 소	Cow
# 사슴	Deer
# 사람	Human 

모델링 1단계) 탐색

* 인터페이스, 추상클래스, 클래스를 찾아보자.
  생물, 동물, 식물, 초식동물, 육식동물, 호랑이, 사자, 소, 사슴, 사람
  
* 데이터를 찾아보자.
  이름, 나이, 풀, 고기
  
* 기능을 찾아보자
  숨을쉬다, 먹다, 자다

모델링 2단계) 분류

* 생물은 동물과 식물로 나뉜다.
* 동물은 초식동물과 육식동물로 나뉜다.
* 초식동물은 소와 사슴으로 나뉜다.
* 육식동물은 호랑이와 사자로 나뉜다.
* 데이터는 이름, 나이, 풀, 고기로 나뉜다.
* 기능은 숨을쉬다, 먹다, 자다로 나뉜다.

모델링 3단계) 정의

* 생물은 추상클래스로 정의한다
- 생물은 모두 이름과 나이를 가지며 기능을 선택적으로 가진다.
- 생물은 공통 기능인 "숨을쉬다" 인터페이스를 구현한다.

* 동물과 식물은 생물을 상속받는 추상클래스로 정의한다
- 동물과 식물은 생물이며 기능을 구체화할 수 없다.
- 동물은 추가로 "먹는다" 인터페이스와 "잔다" 인터페이스를 구현한다.

* 초식동물과 육식동물은 "먹는다" 인터페이스를 다르게 구현하는 동물일 뿐이다.
- 초식을 하는 기능과 육식을 하는 기능은 "먹는다" 인터페이스의 하위 인터페이스이다.

* 호랑이와 사자는 동물이면서 "육식한다" 인터페이스를 구현한다.
  또는 "먹는다" 인터페이스를 구현해서 육식을 해도 좋다.

* 소와 사슴은 동물이면서 "초식한다" 인터페이스를 구현한다.
  또는 "먹는다" 인터페이스를 구현해서 초식을 해도 좋다.

* 사람은 동물이면서 "육식한다"인터페이스와 "초식한다" 인터페이스를 모두 구현한다.
   또는 "먹는다" 인터페이스를 구현해서 육식과 초식을 해도 좋다.
